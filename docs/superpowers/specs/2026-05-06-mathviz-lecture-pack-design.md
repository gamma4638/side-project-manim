## Problem Statement

대학 수학/공학 강의에서 교수는 추상적인 개념을 설명하는 시각 자료를 만들기 어렵고,
학생은 강의 중 막힌 개념을 혼자 이해하기 어렵다.

현재 MathViz는 단일 채팅 페이지로 자연어 입력 → Manim 영상 생성이 가능하지만,
강의 단위 컨텍스트가 없고 생성 결과가 아카이빙되지 않으며 공유가 불가능하다.

---

## What Makes This Cool

- 교수가 PDF 하나 올리면 → Gemini가 핵심 개념 추출 → Manim 영상 여러 개 자동 생성
- 학생들이 그 위에서 추가 질문 → 새 영상 생성 → 전체 아카이브로 누적
- 로그인 없이 UUID URL 하나로 강의 팩 공유 가능
- Google 공모전용: Gemini multimodal (PDF 파싱) 활용 데모에 최적화

---

| 학생 | 강의 팩 URL 접속 → 개념 영상 시청 → 추가 질문 입력 → 새 영상 생성 → 아카이브 공유 |

## 전제 (Confirmed)

1. 공모전 심사위원이 주 시연 대상 — "이런 서비스가 가능하다"를 보여주는 것이 목표
2. 서버 렌더링(5~15초 대기)은 허용 — 마님 퀄리티 우선
3. 로그인/계정 없음 — anonymous 사용, UUID 기반 URL로 접근 제어

## 유저 플로우

### 교수 플로우

```
[랜딩] → [/create] → PDF 업로드
  → Gemini 파싱 (업로드 → 텍스트 추출 → 개념 분류)
  → 핵심 개념 목록 표시 (체크박스로 편집 가능)
  → [생성 시작] 클릭
  → 개념별 Manim 영상 batch 생성 (SSE 진행률 스트리밍)
  → 강의 팩 완성 → /pack/:id URL 생성 → 클립보드 복사
  → 학생에게 URL 공유
```

### 학생 플로우

```
[교수에게 URL 수신] → /pack/:id 접속
  → 핵심 개념 영상 카드 탐색
  → 영상 클릭 → inline 재생 + 설명 텍스트
  → "더 궁금한 게 있으신가요?" 프롬프트 입력
      (강의 컨텍스트 자동 주입 후 Gemini 파싱)
  → 새 Manim 영상 생성 → Q&A 아카이브 섹션에 추가
  → 다른 학생들이 동일 페이지에서 Q&A 영상 공유
```

## Manim 영상 생성 전략

기존 `SCENE_REGISTRY`는 11개 고정 scene만 지원한다. Gemini가 추출한 개념은 이 범위를 벗어날 수 있으므로 아래 2단계 매핑 전략을 사용한다.

**1단계: SCENE_REGISTRY 매핑**

Gemini 개념 추출 시 `topic_id` 필드를 함께 반환하도록 프롬프트에 지시:
```
{
  "concepts": [
    {
      "title": "고유값과 고유벡터 정의",
      "topic_id": "eigenvalue",        ← SCENE_REGISTRY 키
      "params": {"matrix_2x2": [[2,1],[1,2]]},
      "description": "..."
    }
  ]
}
```
`topic_id`가 SCENE_REGISTRY에 있으면 기존 Manim Scene 직접 렌더링.

**2단계: 폴백 — Gemini 코드 생성**

`topic_id`가 SCENE_REGISTRY에 없거나 `"unknown"`인 경우:
- Gemini에게 개념 설명 + Manim Python 코드 생성 요청
- 생성된 코드를 `/output/packs/:id/scenes/concept-{id}.py`로 저장
- subprocess로 `manim -ql concept-{id}.py GeneratedScene` 실행
- 보안: 생성 코드는 별도 subprocess에서만 실행 (FastAPI 프로세스 직접 exec 금지)
- 실패 시: 해당 개념 카드에 "생성 실패" 표시, 배치 나머지 계속 진행

**데모 권장**: 데모용 샘플 팩은 SCENE_REGISTRY 내 topic만 사용하여 실패 리스크 제거.

## Gemini Files API 만료 처리

Gemini Files API 업로드 파일은 **48시간 후 자동 만료**된다. 학생 질문(`/ask`) 처리 시 원본 PDF가 필요하므로 아래 전략을 사용한다.

- `/api/create/upload` 완료 후 원본 PDF를 서버 로컬에 보관: `/output/packs/:id/source.pdf`
- `pack.json`에 `gemini_file_uri` 저장 (업로드 성공 시)
- `/api/pack/:id/ask` 호출 시:
  1. `gemini_file_uri`로 Files API 파일 상태 확인
  2. 유효하면 그대로 사용
  3. 만료됐으면 `/output/packs/:id/source.pdf`를 재업로드 후 새 URI 사용
- **데모 제약 명시**: 팩 생성 후 48시간 내 시연 권장. Railway 재시작 시 로컬 파일 소실 가능 — 중요 데모는 시연 직전 생성.

## 학생 질문 프롬프트 템플릿

`POST /api/pack/:id/ask` 내부에서 기존 `parser.py`의 SYSTEM_PROMPT를 대체하여 아래 컨텍스트를 주입한다:

```
[SYSTEM]
당신은 대학 수학/공학 강의 시각화 도우미입니다.
아래 강의 컨텍스트를 바탕으로 학생의 질문을 분석하고,
가장 적합한 Manim 시각화 주제와 파라미터를 JSON으로 반환하세요.

강의 제목: {pack.title}
핵심 개념 목록:
- {concept_1.title}: {concept_1.description}
- {concept_2.title}: {concept_2.description}
...

[USER]
{student_question}

[RESPONSE FORMAT]
{
  "topic_id": "eigenvalue",        // SCENE_REGISTRY 키 또는 "generated"
  "params": {...},
  "explanation": "한국어 설명 2-3문장",
  "confidence": 0.85
}
```

`confidence < 0.6`이면 "이 강의 내용과 관련된 수학 질문을 입력해주세요." 반환.

## SSE 이벤트 스키마

`GET /api/create/progress/:jobId` 이벤트 형식:

```
data: {"concept_id": "concept-1", "status": "rendering", "progress": 67}
data: {"concept_id": "concept-1", "status": "done", "progress": 100, "video_url": "/output/packs/..."}
data: {"concept_id": "concept-2", "status": "pending", "progress": 0}
data: {"type": "complete", "pack_url": "/pack/uuid-v4"}
```

`status` 값: `pending` | `rendering` | `done` | `failed`

## 새 API 엔드포인트

```

GET  /api/create/progress/:jobId SSE 스트리밍 — 개념별 진행률
GET  /api/pack/:id               팩 메타데이터 + 영상 목록 조회

```
